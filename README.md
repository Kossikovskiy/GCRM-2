# CRM-система на FastAPI

Это бэкенд CRM-системы, разработанный на Python с использованием фреймворка FastAPI. Приложение развернуто на Linux-сервере в качестве системной службы (systemd) и использует PostgreSQL в качестве базы данных.

## Список изменений

### Аутентификация
- Заменена самописная JWT-аутентификация (passlib/bcrypt) на Auth0 с входом через Яндекс.
- Внедрены белые списки для управления доступом.

### База данных
- Выполнена миграция с SQLite на PostgreSQL (через Supabase).
- Добавлен собственный кэш в памяти (`_Cache`) с TTL=5 минут и инвалидацией при мутациях (эндпоинт `POST /api/cache/invalidate`). Это обеспечивает быстрый ответ и снижает нагрузку на БД.
- Добавлена модель `Budget` для ведения бюджета.

### Новые возможности API (v13.0)
- **Аналитика:** Добавлен эндпоинт `/api/analytics/funnel` для построения воронки продаж.
- **Бюджетирование:** CRUD-эндпоинты для управления бюджетом (`/api/budget`).
- **Экспорт данных:**
    - `/api/export/excel`: Экспорт сделок, расходов, аналитики и бюджета в `.xlsx` файл.
    - `/api/export/pdf`: Экспорт годового отчета с инфографикой в `.pdf` файл.
- Добавлен эндпоинт `/api/years`.
- Добавлена поддержка фильтрации по году для сделок, расходов и задач.
- Фронтенд (`index.html`) уже адаптирован.

### Визуальные улучшения
- **Шрифты:** Playfair Display (заголовки) и DM Sans (основной текст).
- **KPI-карточки:** с цветными полосами, тенями и hover-эффектами.
- **Шапка:** с анимированным подчёркиванием активного пункта.
- **Таблицы:** с шапками, бейджами статусов, тегами категорий и денежным форматированием.
- **Канбан:**
    - Отображение итоговой суммы в столбце.
    - Заглушки для пустых колонок.
    - Переход на grid-вёрстку (`auto-fit`) для равномерного заполнения ширины.
- **Экран входа:** двойная рамка, декоративное правило, анимация стрелки.
- Полностью переработана тёмная тема.

### Drag & Drop карточек
- Реализован нативный HTML5 DnD без библиотек.
- При перетаскивании карточка становится полупрозрачной, колонка подсвечивается, появляется плейсхолдер.
- После сброса отправляется `PATCH /deals/{id}/stage`; в случае ошибки карточка возвращается на место.

### Добавление задач
- Кнопка «＋ Новая задача» в шапке страницы задач.
- Модальное окно с полями «Название» и «Срок».
- Отправка `POST /tasks` и обновление списка.
- Закрытие по `Esc` или клику вне модалки.

## 🤖 Телеграм-бот (`bot.py`)

В проекте есть телеграм-бот, который ежедневно отправляет в заданный чат сводку о состоянии дел.

### Функции бота:

1.  **Утренний отчет** (будние дни, 09:00): 
    *   Показывает задачи на сегодня.
    *   Напоминает о сделках в активных стадиях.
    *   Предупреждает о необходимости ТО для техники.

2.  **Вечерний отчет** (ежедневно, 18:00):
    *   Дублирует информацию из утреннего отчета.
    *   Добавляет задачи, запланированные на *завтра*.
    *   В конце присылает случайную мотивирующую цитату или хокку.

### Настройка

1.  **Укажите данные в `bot.py`**:
    ```python
    TG_TOKEN = "ВАШ_ТЕЛЕГРАМ_ТОКЕН"
    TG_CHAT = "ID_ВАШЕГО_ЧАТА" 
    ```

2.  **Настройте Cron**: Задачи для `cron` уже настроены и запускают бота в нужное время. Управлять ими можно через `crontab -e`.

### Ручной запуск

Вы можете запустить отправку отчета вручную в любое время:

```bash
# Активировать виртуальное окружение
source /var/www/crm/venv/bin/activate

# Запустить скрипт
python /var/www/crm/GCRM-2/bot.py
```

## 💾 Резервное копирование (`backup.sh`)

Для обеспечения сохранности данных настроено автоматическое создание резервных копий базы данных PostgreSQL.

### Как это работает:

*   **Скрипт**: `backup.sh` в корне проекта.
*   **Планировщик**: `cron` запускает скрипт ежедневно.
*   **Время запуска**: 03:00 ночи.
*   **Место хранения**: `/var/www/crm/GCRM-2/backups/` (папка исключена из Git).
*   **Формат**: Сжатый SQL-дамп (`.sql.gz`).
*   **Ротация**: Скрипт автоматически удаляет бэкапы старше 1 дня, **храня только 2 последние копии** (за сегодня и за вчера).

### Ручное создание бэкапа

Если нужно создать бэкап немедленно:

```bash
# Запустить скрипт напрямую
/var/www/crm/GCRM-2/backup.sh
```

### Восстановление из бэкапа

Для восстановления базы данных из файла `.sql.gz`:

1.  **Загрузите** нужный файл бэкапа на сервер.
2.  **Выполните команду**, указав путь к файлу и URL вашей базы данных:

```bash
gunzip < /путь/к/вашему/backup-file.sql.gz | psql -d '''ВАШ_DATABASE_URL'''
```

> **Внимание!** Команда восстановления полностью перезапишет текущее состояние базы данных данными из файла.

## Развертывание (Production на Linux)

Это руководство описывает, как было настроено приложение для работы на сервере.

### 1. Расположение файлов

Проект расположен в директории `/var/www/crm/GCRM-2/`. Виртуальное окружение находится в `/var/www/crm/venv/`.

### 2. Скрипт запуска (`start.sh`)

Для корректного запуска приложения из-под `systemd` используется bash-скрипт, который активирует виртуальное окружение.

**`start.sh`:**
```bash
#!/bin/bash
source /var/www/crm/venv/bin/activate
/var/www/crm/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000
```
*Скрипту необходимо дать права на исполнение (`chmod +x start.sh`).*

### 3. Служба `systemd`

Приложением управляет `systemd`. Конфигурационный файл службы находится в `/etc/systemd/system/crm.service`.

**`crm.service`:**
```ini
[Unit]
Description=Grass CRM FastAPI
After=network.target

[Service]
User=root
Group=www-data
WorkingDirectory=/var/www/crm/GCRM-2
ExecStart=/var/www/crm/GCRM-2/start.sh
Restart=always

[Install]
WantedBy=multi-user.target
```

### 4. Управление службой

Основные команды для управления сервисом:
- `systemctl daemon-reload`      # Перечитать конфигурацию после изменений в .service файле
- `systemctl enable crm.service`     # Включить автозапуск при старте системы
- `systemctl start crm.service`      # Запустить сервис
- `systemctl restart crm.service`    # Перезапустить сервис
- `systemctl status crm.service`     # Проверить статус сервиса

## Локальная разработка

### 1. Установка
```bash
# Клонировать репозиторий
git clone [URL_РЕПОЗИТОРИЯ]
cd GCRM-2

# Создать и активировать виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

### 2. Настройка `.env`
Создайте файл `.env` в корне проекта и укажите в нем необходимые переменные, как минимум URL для подключения к базе данных.
```
DATABASE_URL="postgresql://user:password@host:port/dbname"
TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН"
TELEGRAM_CHAT_ID="ВАШ_ЧАТ_ID"
```

### 3. Запуск
```bash
# Запустить приложение в режиме разработки
uvicorn main:app --reload --port 8000
```
API будет доступно по адресу `http://127.0.0.1:8000`. Автоматическая документация Swagger UI находится по адресу `http://127.0.0.1:8000/docs`.
