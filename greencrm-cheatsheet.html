🔌 Проблемы с портом 8000
Кто занимает порт?
Показывает PID процесса на порту 8000
fuser 8000/tcp
Принудительно освободить порт
Убить всё что висит на 8000 (потом запускать crm вручную)
fuser -k 8000/tcp
Все процессы uvicorn
ps aux | grep uvicorn
Убить конкретный процесс
Заменить 12345 на нужный PID из ps aux
kill -9 12345
Симптом «address already in use»: запусти fuser -k 8000/tcp && sleep 2 && systemctl start crm

🚑 Типичные проблемы и решения
Симптом → Причина → Решение

❌ address already in use
Старый процесс не умер, занимает порт 8000
fuser -k 8000/tcp && sleep 2 && systemctl start crm


❌ status=203/EXEC
start.sh не исполняемый (нет прав x)
chmod +x /var/www/crm/GCRM-2/start.sh && systemctl restart crm


❌ Assignment outside of section
В crm.service попал мусор (заголовок nano)
tee /etc/systemd/system/crm.service << 'EOF' [Unit] Description=Grass CRM FastAPI After=network.target [Service] User=root WorkingDirectory=/var/www/crm/GCRM-2 EnvironmentFile=/var/www/crm/GCRM-2/.env ExecStart=/var/www/crm/GCRM-2/start.sh TimeoutStopSec=15 [Install] WantedBy=multi-user.target EOF systemctl daemon-reload && systemctl restart crm


❌ JSON parse error в браузере
Сервер упал, вернул HTML вместо JSON. Смотреть логи и перезапускать
journalctl -u crm.service -n 50 --no-pager


❌ 401 / редирект на логин
Сессия истекла. Просто войти заново через браузер
